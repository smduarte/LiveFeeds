SIMULATORS

  • SimSim Simulator
    • A discrete event simulator, implemented in Java, developed during the early stages of project. Over the duration of the project, SimSim has been continuously updated to allow the fast prototyping of distributed algorithms, with emphasis on easy of programming and detailed network traffic monitoring.
    • Chord DHT Example <will launch in a separate window; needs java 1.6 installed> (executable jar)
    • Java Source
  • LiveFeeds Simulador
    • Implementations of the algorithms that provide the LiveFeeds' filtered multicast solution, using the SimSim.
    • LiveFeeds Simulation Example <will launch in a separate window; needs java 1.6 installed> (executable jar)
      • This simulation shows the state of the membership substrate (Catadupa) and its communication costs. Moreover, its plots performance characteristics for Turmoil, the filtered multicast algorithms, evaluation the cost in terms of CPU and message routing depending on the popularity of the events.
      • Java Source


PROTOTYPES

  •  Meeds/Feeds Platform
    • A event dissemination platform with support for mobility. Event notification employs the LiveFeeds filtered multicast algorithms as the basis for a content-based publish/subscribe subtrate.
      • Application Example <executable jar; needs java 1.6 installed>
      • Java Source
  • 4Sensing Platform - Decentralized processing of Participatory Sensing data.
    • Traffic Monitoring Application Simulator <executable jar; needs java 1.6 installed>
    • Java/Groovy Source
  • CollFeeds - LiveFeeds prototype
    • Implemented as a HTTP proxy, allowing RSS feeds to be pushed among clients in a collaborative manner.
    • Java Source + Instructions
